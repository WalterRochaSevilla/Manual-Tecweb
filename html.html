Elemento/Etiqueta Descripción Uso
<a>: El elemento ancla El elemento HTML <a> (o elemento ancla), con su atributo href, crea un hipervínculo a páginas web, archivos, direcciones de correo electrónico, ubicaciones en la misma página o cualquier otra cosa que una URL pueda abordar. El contenido dentro de cada <a> debe indicar el destino del enlace. Si el atributo href está presente, presionar la tecla Intro mientras se enfoca en el elemento <a> lo activará.
<abbr>: El elemento abreviatura El elemento HTML <abbr> es un Abreviacion o Acronimo. Cuando se usa una abreviatura y desea proporcionar una expansión o definición fuera del flujo del contenido del documento, use <abbr>con un título apropiado. Para definir una abreviatura que puede ser desconocida para el lector, presente el término usando <abbr> y texto en línea que proporcione la definición. Incluya un atributo de título solo cuando la expansión o definición en línea no esté disponible. Cuando es necesario anotar semánticamente la presencia de una abreviatura en el texto, el elemento <abbr> es útil. Esto se puede utilizar, a su vez, con fines de estilo o secuencias de comandos. Puede utilizar <abbr> junto con <dfn> para establecer definiciones de términos que son abreviaturas o acrónimos.
<address>: El elemento Dirección de contacto El elemento HTML <address> indica que el HTML adjunto proporciona información de contacto para una persona o personas, o para una organización.
<article>: El elemento Contenido del artículo El elemento HTML <article> representa una composición independiente en un documento, página, aplicación o sitio, que está destinado a ser distribuible o reutilizable de forma independiente . Cada <article> debe identificarse, normalmente mediante la inclusión de un encabezado (elemento <h1> - <h6>) como elemento secundario del elemento <article>. Cuando se anida un elemento <article>, el elemento interior representa un artículo relacionado con el elemento exterior. Por ejemplo, los comentarios de una publicación de blog pueden ser elementos <article> anidados en el <article> que representa la publicación de blog. La información del autor de un elemento <article> se puede proporcionar a través del elemento <address>, pero no se aplica a los elementos <article> anidados. La fecha y la hora de publicación de un elemento <article> se pueden describir utilizando el atributo datetime de un elemento <time>.
<aside>: El elemento Aside El elemento HTML <aside> representa una parte de un documento cuyo contenido solo está indirectamente relacionado con el contenido principal del documento. Los asides se presentan con frecuencia como barras laterales o cuadros de llamada
<body>: el elemento del cuerpo del documento El elemento HTML <body> representa el contenido de un documento HTML. Solo puede haber un elemento <body> en un documento
<blockquote>: el elemento de cotización en bloque El elemento HTML <blockquote> indica que el texto adjunto es una cita extendida. Por lo general, esto se representa visualmente mediante identacion. Se puede dar una URL para la fuente de la cita usando el atributo cite, mientras que una representación de texto de la fuente se puede dar usando el elemento <cite>.
<br>: El elemento de salto de línea El elemento HTML <br≫ produce un salto de línea en el texto (retorno de carro). Es útil para escribir un poema o un discurso, donde la división de líneas es significativa. No Se debe usar para hacer saltos de linea, el unico uso real de esta etiqueta se dá cuando se escriben poesías, canciones o haikus (Muy romántico por parte de los creadores de esta etiqueta)
<code>: el elemento de código en línea El elemento HTML <code> muestra su contenido con estilo para indicar que el texto es un fragmento corto de código de computadora. De forma predeterminada, el texto del contenido se muestra con la fuente monoespaciada predeterminada del agente de usuario. Se puede utilizar para mostrar codigo
<cite>: el elemento de cita El elemento HTML <cite> se utiliza para marcar el título de un trabajo creativo citado. La referencia puede estar en forma abreviada de acuerdo con las convenciones apropiadas al contexto relacionadas con los metadatos de citas.
<del>: el elemento de texto eliminado El elemento HTML <del> representa un rango de texto que se ha eliminado de un documento. Esto se puede usar al representar "cambios de seguimiento" o información de diferencias de código fuente.
<dfn>: el elemento de definición El elemento HTML <dfn> se utiliza para indicar el término que se define dentro del contexto de una frase u oración de definición. El elemento antepasado <p>, el par <dt>/<dd> o el antepasado <sección> más cercano del elemento <dfn> se considera la definición del término. Si el elemento <dfn> tiene un atributo de <title>, se considera que el valor del atributo de título es el término que se está definiendo. El elemento aún debe tener texto dentro, pero ese texto puede ser una abreviatura (quizás usando <abbr>) u otra forma del término.Si <dfn> contiene un solo elemento secundario y no tiene ningún contenido de texto propio, y el elemento secundario es un elemento <abbr>con un atributo de título en sí mismo, entonces el valor exacto del título del elemento <abbr>es el término que se define.De lo contrario, el contenido de texto del elemento <dfn> es el término que se define. si el elemento <dfn> tiene un atributo de título, debe contener el término que se define y ningún otro texto.
<div>: el elemento de división de contenido El elemento HTML <div> es el contenedor genérico para el contenido de flujo. El elemento <div> debe usarse solo cuando ningún otro elemento semántico (como <article> o <nav>) sea apropiado.
<em>: El elemento Énfasis El elemento HTML <em> marca el texto que tiene énfasis. El elemento <em> se puede anidar, y cada nivel de anidamiento indica un mayor grado de énfasis. Se utiliza el elemento <em> para Enfatizar Partes especificas del texto
<footer> El elemento HTML <footer> representa un pie de página para su ancestro más cercano que secciona el contenido o secciona el elemento raíz. Un <footer> normalmente contiene información sobre el autor de la sección, datos de derechos de autor o enlaces a documentos relacionados.
<h1> - <h6>: Los elementos de encabezado de sección HTML Los elementos HTML <h1> - <h6> representan seis niveles de encabezados de sección. <h1> es el nivel de sección más alto y <h6> es el más bajo. Las aplicaciones de usuario pueden utilizar la información de encabezado para construir automáticamente una tabla de contenido para un documento. No utilice elementos de encabezado para cambiar el tamaño del texto. En su lugar, utilice la propiedad CSS font-size. No omita los niveles de encabezado: comience siempre desde <h1>, seguido de <h2> y así sucesivamente.
<head>: el elemento de metadatos del documento (encabezado) El elemento HTML <head> contiene información legible por máquina (metadatos) sobre el documento. Se utiliza para asignar metadatos, como ser: Nombre del documento, un set de caracteres, o enlazar con hojas de estilos
<header> El elemento HTML <header> representa contenido introductorio, normalmente un grupo de ayudas introductorias o de navegación. El elemento <header> tiene un significado idéntico al de la función de punto de referencia del banner de todo el sitio
<hr>: El elemento de ruptura temática (regla horizontal) El elemento HTML <hr> representa una ruptura temática entre elementos a nivel de párrafo Un uso común podria ser el de un cambio de escena en una historia o un cambio de tema dentro de una sección.
<html>: el documento HTML/elemento raíz El elemento HTML <html> representa la raíz (elemento de nivel superior) de un documento HTML, por lo que también se denomina elemento raíz. Todos los demás elementos deben ser descendientes de este elemento.
<img>: el elemento de inserción de imagen El elemento HTML <img> incrusta una imagen en el documento. Se usa para colocar imagenes,de diferentes tipos ya sean jpg , gifs, etc...
<ins> El elemento HTML <ins> representa un rango de texto que se ha agregado a un documento. Puede usar el elemento <del>para representar de manera similar un rango de texto que se ha eliminado del documento.
<kbd>: el elemento de entrada de teclado El elemento HTML <kbd> representa un tramo de texto en línea que indica la entrada textual del usuario desde un teclado, entrada de voz o cualquier otro dispositivo de entrada de texto. Por convención, el agente de usuario presenta de forma predeterminada el contenido de un elemento <kbd> utilizando su fuente monoespaciada predeterminada, aunque el estándar HTML no lo exige.
<main> El elemento HTML <main> representa el contenido dominante del <body> de un documento. El área de contenido principal consiste en contenido que está directamente relacionado o amplía el tema central de un documento o la funcionalidad central de una aplicación. Un documento no debe tener más de un elemento <main> que no tenga especificado el atributo oculto.
<mark>: El elemento Marcar Texto El elemento HTML <mark> representa texto que está marcado o resaltado para fines de referencia o notación debido a la relevancia del pasaje marcado en el contexto que lo contiene. Se utiliza como si se tratas3e de un resaltador, (basicamente para dar enfasis en una parte del texto, no confindir con <em>)
<meta>: el elemento de metadatos El elemento HTML <meta> representa metadatos que no pueden ser representados por otros elementos meta-relacionados de HTML Algunos ejemplos como <base>, <link>, <script>, <style> o <title>
<p>: El elemento Párrafo El elemento HTML <p> representa un párrafo. Los párrafos generalmente se representan en los medios visuales como bloques de texto separados de los bloques adyacentes por líneas en blanco y/o sangría en la primera línea. Los párrafos HTML pueden ser cualquier agrupación estructural de contenido relacionado.
<pre>: El elemento de texto preformateado El elemento HTML <pre> representa texto preformateado que debe presentarse exactamente como está escrito en el archivo HTML. El texto normalmente se representa con una fuente no proporcional o monoespaciada. Los espacios en blanco dentro de este elemento se muestran tal como están escritos.
<q>: el elemento de cotización en línea El elemento HTML <q> indica que el texto adjunto es una cita breve en línea. La mayoría de los navegadores modernos implementan esto encerrando el texto entre comillas. Este elemento está diseñado para citas cortas que no requieren saltos de párrafo; para citas largas use el elemento <blockquote>.
<s>: el elemento tachado El elemento HTML <s> muestra el texto tachado o con una línea que lo atraviesa. Usa el elemento <s> para representar cosas que ya no son relevantes o precisas. Sin embargo, <s> no es apropiado para indicar ediciones de documentos; para ello, utilice los elementos <del> e <ins>, según corresponda.
<samp>: el elemento de salida de muestra El elemento HTML <samp> se utiliza para encerrar texto en línea que representa la salida de muestra (o citada) de un programa de computadora. Su contenido generalmente se representa utilizando la fuente monoespaciada predeterminada del navegador (como Courier o Lucida Console)
<
>: El elemento Sección Genérica		 El elemento HTML <section> representa una sección independiente genérica de un documento. Solo debe usarse si no hay un elemento más específico para representarlo.
<small>: el elemento de comentario lateral El elemento HTML <small> representa comentarios secundarios y letra pequeña, Se Puede Usar como derechos de autor y texto legal, independientemente de su estilo de presentación. De forma predeterminada, representa el texto dentro de un tamaño de fuente más pequeño, como de pequeño a x-pequeño.
<span>: el elemento de extensión de contenido El elemento HTML <span> es un contenedor genérico en línea para redactar contenido, que no representa inherentemente nada. Se puede usar para agrupar elementos con fines de estilo (usando los atributos class o id), o porque comparten valores de atributo, como lang. Debe usarse solo cuando ningún otro elemento semántico sea apropiado. <span> es muy parecido a un elemento <div>, pero <div> es un elemento a nivel de bloque mientras que <span> es un elemento en línea.
<strong>: El elemento de gran importancia El elemento HTML <strong> indica que su contenido tiene una gran importancia, seriedad o urgencia.

El elemento <strong> es para contenido que es de "gran importancia", incluidas cosas de gran seriedad o urgencia
(como advertencias). Esta podría ser una oración que sea de gran importancia para toda la página,
o simplemente podría tratar de señalar que algunas palabras son de mayor importancia en comparación con el contenido cercano.
x Normalmente, este elemento se representa de forma predeterminada con un peso de fuente en negrita. Sin embargo,
no debe usarse para aplicar un estilo audaz; use la propiedad CSS font-weight para ese propósito.
Utilice el elemento <b> para llamar la atención sobre un texto determinado sin indicar un mayor nivel de importancia.
Use el elemento <em> para marcar el texto que tiene énfasis.

Otro uso aceptado para <strong> es para indicar las etiquetas de los párrafos que representan notas o advertencias dentro del texto de una página.
style: el elemento de información de estilo El elemento HTML style contiene información de estilo para un documento o parte de un documento. Contiene CSS, que se aplica al contenido del documento que contiene el elemento style
<sub>: El elemento Subíndice El elemento HTML <sub> especifica el texto en línea que debe mostrarse como subíndice únicamente por motivos tipográficos. Los subíndices generalmente se representan con una línea de base más baja usando texto más pequeño. Ampliamente usado para la escritura de elementos químicos
<sup>: El elemento Superíndice El elemento HTML <sup> especifica el texto en línea que se mostrará como superíndice únicamente por motivos tipográficos. Los superíndices generalmente se representan con una línea de base elevada usando texto más pequeño. Utilizados mayormente para la escritura de formulas fisico/matematicas
<table>: El elemento Tabla El elemento HTML <table> representa datos tabulares, es decir, información presentada en una tabla bidimensional compuesta por filas y columnas de celdas que contienen datos.
<tbody>: el elemento del cuerpo de la tabla El elemento HTML <tbody> encapsula un conjunto de filas de tabla (elementos <tr>), lo que indica que comprenden el cuerpo de la tabla (<table>) El elemento <tbody>, junto con sus elementos <thead> y <tfoot> relacionados, proporcionan información semántica útil que se puede utilizar al renderizar para pantalla o impresora.
<td>: El elemento Celda de datos de la tabla El elemento HTML <td> define una celda de una tabla que contiene datos. Participa en el modelo de tabla. Se puede utilizar para hacer tablas.
<tfoot>: El elemento Pie de tabla El elemento HTML <tfoot> define un conjunto de filas que resumen las columnas de la tabla. Se puede utilizar para hacer tablas.
<th>: el elemento de encabezado de tabla El elemento HTML <th> define una celda como encabezado de un grupo de celdas de tabla. La naturaleza exacta de este grupo está definida por los atributos de alcance y encabezados. Se puede utilizar para hacer tablas.
<thead>: el elemento de encabezado de tabla El elemento HTML <thead> define un conjunto de filas que definen el encabezado de las columnas de la tabla. Se puede utilizar para hacer tablas
<title>: el elemento Título del documento El elemento HTML <title> define el título del documento que se muestra en la barra de título del navegador o en la pestaña de una página. Solo contiene texto; las etiquetas dentro del elemento se ignoran.
<tr>: El elemento Fila de la tabla El elemento HTML <tr> define una fila de celdas en una tabla Las celdas de la fila se pueden establecer usando una combinación de elementos <td> (celda de datos) y <th> (celda de encabezado)
<ul>: El elemento Lista desordenada El elemento HTML <ul> representa una lista desordenada de elementos, normalmente representada como una lista con viñetas El elemento <ul> es para agrupar una colección de elementos que no tienen un orden numérico, y su orden en la lista no tiene sentido.
<wbr>: El elemento Oportunidad de salto de línea El elemento HTML <wbr> representa una oportunidad de salto de palabra: una posición dentro del texto donde el navegador puede opcionalmente dividir una línea, aunque sus reglas de división de línea no crearían de otro modo una ruptura en esa ubicación. Se utiliza para Enseñar al "Compilador" donde se puede recortar la url, al momento de que se tenga que hacer el salto de linmea necesario para mostrar todo el texto en la pantalla
